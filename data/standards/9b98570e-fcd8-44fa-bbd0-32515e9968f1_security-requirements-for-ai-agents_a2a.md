# Security Requirements for AI Agents (A2A)

**Status:** Draft | **Organization:** IETF

## Description
This document specifies security requirements for AI agents across various interaction stages including provisioning, registration, discovery, and cross-domain interconnection. It focuses on establishing a secure framework for agent-to-agent (A2A) communication and access control.

## Latest Summary
This informational Internet-Draft outlines the security architecture and requirements for autonomous AI agents, specifically focusing on multi-agent systems and cross-domain interactions. As AI agents increasingly handle sensitive tasks across various business scenarios, the document identifies critical security gaps in current deployments. The proposed architecture introduces four primary components: a **Firewall** for traffic control, a **Master Agent** for central orchestration and policy management, individual **Agents**, and **Heterogeneous Systems** (databases, APIs, etc.).

The draft categorizes security requirements into three main sequential interaction stages:
1.  **Provisioning, Registration, and Discovery:** Establishing initial trust through hardware roots of trust (TPM/HSM) or manufacturer credentials (IDevID). It introduces the **Agent Credential Authority (ACA)** for issuing X.509 or token-based identities and the **Agent Registry Service (ARS)** for discovery. A key requirement includes the use of an **AI Bill of Materials (AIBOM)** to inventory models and datasets.
2.  **Cross-domain Interconnection:** Focusing on secure transmission and authenticated external calls between different trust domains.
3.  **Access Control:** Implementing zero-trust principles where the Master Agent validates access tokens and manages credentials for heterogeneous internal systems.

The document emphasizes that AI agents should not have direct access to long-term secrets due to prompt injection risks; instead, they should utilize platform-level secret managers and temporary access tokens. It also aligns with existing IETF work, specifically the **WIMSE** (Workload Identity in Multi-Service Environments) group, for identity management.

## What's New
The latest update (version 01, published February 2026) expands on the specialized components for agent security. It introduces the concepts of the **Agent Credential Authority (ACA)** and **Agent Registry Service (ARS)**. Notably, it adds a requirement for an **AI Bill of Materials (AIBOM)** during the credential request phase to ensure the integrity of the agent’s supply chain, including models and datasets. The update also explicitly addresses the risk of **Prompt Injection**, mandating that agents use secure intermediaries or platform-level secret managers rather than holding direct access to secrets. Furthermore, it strengthens the alignment with the **WIMSE** (Workload Identity in Multi-Service Environments) initiative for AI agent identity mapping.
