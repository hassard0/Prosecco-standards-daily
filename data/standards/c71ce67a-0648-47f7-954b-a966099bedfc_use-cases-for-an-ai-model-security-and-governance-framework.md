## Latest Summary
This Internet-Draft proposes the integration of **AI Network Security Agents** into core network devices like routers to evolve security from passive defense to an intelligent, adaptive framework. Traditional network security (ACLs, signature-based detection) is criticized for its inability to handle modern threats such as zero-day vulnerabilities and DDoS attacks due to manual configuration delays and high false-positive rates.

The framework proposes three core capabilities: **adaptive learning**, **automation**, and **distributed collaboration**. By leveraging machine learning, behavioral analysis, and real-time data fusion, routers can transform into active security orchestrators.

Key use cases identified include:
1.  **Dynamic Defense**: Establishing "normal behavior baselines" to detect unknown threats, tracing attack chains using knowledge graphs, and predicting potential attack surfaces.
2.  **Intelligent Policy Management**: Translating natural language intent into ACL rules, dynamically optimizing traffic policies (e.g., QoS adjustments during peak hours), and detecting logic conflicts in existing rules.
3.  **Device Configuration Security**: Automating regular compliance checks against security templates and simulating the impact of configuration changes before deployment to verify functionality.
4.  **Global Collaborative Defense**: Aggregating threat intelligence from diverse sources (e.g., CISA, VirusTotal) and orchestrating linked responses between upstream and downstream devices to block lateral movement of threats across network segments.

## What's New
The initial draft (version 00) was introduced on November 1, 2025. This document marks the beginning of a proposed standards track effort within the Routing Area Working Group (RTGWG). It focuses on shifting the role of routers from passive traffic handlers to proactive security orchestrators by integrating AI agents capable of intent-based policy translation, real-time behavioral baseline creation, and collaborative cross-device threat response. Currently, 'Security Considerations' and 'IANA Considerations' are placeholders (TBD), indicating the framework is in its earliest conceptual stage.
