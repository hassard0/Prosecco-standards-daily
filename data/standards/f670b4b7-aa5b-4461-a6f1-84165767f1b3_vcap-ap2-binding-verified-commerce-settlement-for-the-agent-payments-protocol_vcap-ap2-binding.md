# VCAP-AP2 Binding: Verified Commerce Settlement for the Agent Payments Protocol (VCAP-AP2 Binding)

**Status:** Emerging | **Organization:** IETF

## Description
This specification defines how the Verified Commerce for Agent Protocols (VCAP) settlement layer binds to the Agent Payments Protocol (AP2). It provides the cryptographic proof bindings and verification mechanisms required for escrow fund release based on delivery confirmation in agent-initiated transactions.

## Latest Summary
The **VCAP-AP2 Binding** is an informational Internet-Draft that defines the integration between the **Verified Commerce for Agent Protocols (VCAP)** settlement layer and the **Agent Payments Protocol (AP2)**. While AP2 handles payment intents and mandates for AI agent transactions, it lacks a detailed mechanism for "delivery-confirmation" and escrow release. This specification fills those technical gaps by mapping AP2 fields to VCAP's escrow state machines and cryptographic proof systems.

Key decisions in this binding include the definition of the `authorized` to `captured` state transition based on VCAP verification callbacks. It introduces a new evidence type for AP2—`vcap.verification_proof`—which includes SHA-256 hashes and HMAC signatures to ensure proof integrity. This allows for automated, verifiable settlement where funds are only captured once a delivery is confirmed by a verifier (such as browser automation or LLM evaluation), strictly within a defined dispute window.

Open questions often surround the specific implementations of "verifiers" and the handling of timeouts, which the draft addresses by suggesting manual reviews or escalation to VCAP's internal logic. The standard is designed to ensure that agent-to-agent commerce remains trustless and cryptographically sound across different payment rails.

## What's New
The version **-00** draft represents the first formal specification of the VCAP-AP2 binding. It introduces a comprehensive mapping between AP2's `PaymentIntent` and VCAP's `escrow_hold`. Notably, it defines new AP2 webhook events for verification (initiated, completed, and timeout) and provides a cryptographic structure for delivery confirmation that was previously undefined in the core AP2 protocol. It also includes an extensible evidence object to store VCAP-specific verification proofs within AP2 transaction metadata.
