# Internet of Agents Task Protocol (IoA Task Protocol) for Heterogeneous Agent Collaboration (IoA Task Protocol)

**Status:** Draft | **Organization:** IETF

## Description
A protocol designed to support distributed, heterogeneous agent collaboration in intelligent systems. It enables dynamic team formation, adaptive task coordination, and structured communication among agents with diverse architectures and tools.

## Latest Summary
The **Internet of Agents Task Protocol (IoA Task Protocol)** is an emerging IETF Standards Track draft designed to facilitate collaboration within distributed, heterogeneous multi-agent systems. As artificial intelligence evolves from single-agent models to collaborative networks, this protocol addresses the interoperability gaps between diverse agent architectures, tools, and execution environments (such as AutoGPT or Open Interpreter).

The protocol introduces a layered architecture implemented symmetrically on both the server and client sides. The server-side focuses on global coordination, agent discovery, and group management, while the client-side handles local task execution and team participation. Key features include an **Agent Registry Block** for semantic capability searches, a **Team Formation Block** for dynamic group creation, and a **Session State Machine** to manage dialogue states—ranging from discussion and task assignment to conclusion.

Architecturally, the IoA Task Protocol is designed to operate across diverse network infrastructures, including edge-cloud systems and future 6G mobile networks. It aims to solve common limitations in current frameworks, such as rigid workflows and the inability to deploy agents across different physical devices. By leveraging structured message formats and decentralized deployment capabilities, it supports large-scale intelligent collaboration in sectors like healthcare, transportation, and human-AI teaming.

## What's New
The latest update (**draft-02**, released January 2026) significantly expands on the protocol's positioning within the network layering system, particularly emphasizing its relevance to **6G and edge-cloud infrastructures**. It formalizes the **Session State Machine** to govern collaboration states and introduces specific methods for **Autonomous Team Formation**. The update also clarifies the protocol's relationship with the A2A (Agent-to-Agent) protocol and adds considerations for future enhancements in semantic interoperability and ontology alignment. Recent activity highlights a shift toward supporting decentralized deployment across diverse devices, moving beyond single-device simulations.
