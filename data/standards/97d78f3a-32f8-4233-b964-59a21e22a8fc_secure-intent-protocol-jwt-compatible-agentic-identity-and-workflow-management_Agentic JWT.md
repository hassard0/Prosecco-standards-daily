# Secure Intent Protocol: JWT Compatible Agentic Identity and Workflow Management (Agentic JWT)

**Status:** Draft | **Organization:** IETF

## Description
An extension to OAuth 2.0 and JWT designed to address authorization challenges unique to autonomous agentic AI systems. It introduces cryptographic agent identity through checksums and workflow-aware token binding to ensure alignment between user intent and autonomous agent actions.

## Latest Summary
The **Secure Intent Protocol (Agentic JWT)** is an extension to the OAuth 2.0 framework designed to address specific security risks posed by autonomous AI agents. Traditional authorization models assume a deterministic relationship between a user's intent and a client application's actions. However, in agentic AI systems, a "non-deterministic" model may generate sub-agents or dynamic workflows that deviate from the original user intent—a phenomenon termed the **"intent-execution separation problem."**

This protocol introduces a "Zero-Trust" architecture for AI agents by requiring cryptographic verification of an agent's identity and its specific workflow. Key components include **Agent Checksums**, which are unique identifiers derived from an agent's system prompts, toolsets, and configurations. By binding these checksums to the authorization process, the protocol prevents unauthorized agents from assuming the privileges of a human user or other high-privileged agents. 

Furthermore, the standard introduces a new **OAuth 2.0 grant type (`agent_checksum`)** and mechanisms for workflow-aware token binding. This ensures that a token issued to an agent is valid only for the specific, pre-authorized steps of an AI-driven task. It also incorporates a version of **Proof-of-Possession (PoP)** at the agent level to mitigate the risk of token replay by separate malicious agents running within the same multi-agent environment. The specification maintains backward compatibility with existing OAuth 2.0 infrastructure while providing the granularity needed for modern AI orchestration.

## What's New
The protocol is currently in its first iteration (**draft-00**), released on **December 31, 2024**. Recent updates focus on defining the core architectural components for agentic identity. 

Most notably, the draft introduces the **`agent_checksum` grant type**, which allows an Authorization Server to validate an agent's "DNA"—consisting of its system prompts, available tools, and underlying code—before issuing a token. The update also includes a detailed **sequence hash computation** method to link multi-agent delegation chains, ensuring that when a supervisor agent delegates a task to a sub-agent, the cryptographic "intent" remains intact and verifiable across the execution flow. Security analysis has been expanded to include a **threat matrix** specifically targeting AI-related vulnerabilities like prompt injection used for privilege escalation.
