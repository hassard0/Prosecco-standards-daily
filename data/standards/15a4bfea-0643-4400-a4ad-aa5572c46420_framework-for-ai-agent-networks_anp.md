# Framework for AI Agent Networks (ANP)

**Status:** Draft | **Organization:** IETF

## Description
This document defines a framework for AI agent networks based on the Agent Network Protocol (ANP). It specifies the basic functions required to support AI agent communication within a trusted domain.

## Latest Summary
The "Framework for AI Agent Networks" is an IETF Internet-Draft that defines a structured approach for communication and collaboration between AI agents. The framework is built upon the **Agent Network Protocol (ANP)** and aims to address the growing need for standardized interoperability in multi-agent systems. It specifically targets agents managed within an operator's network, including terminal-side and network-side agents, while excluding cross-domain communication from its current scope.

The framework identifies three primary roles: **AI Agents**, which are autonomous entities executing tasks; **Agent Registration Servers**, which handle capability registration and discovery; and **Agent Communication Servers**, which manage sessions and routing. Key technical components described include digital identifiers for unique agent addressing, multimodal communication support, and agent description documents in both natural language and structured formats. The protocol workflow supports two primary communication options: mediated communication via a server or direct point-to-point interaction between agents after discovery.

Architecturally, the document covers essential network functions such as task coordination (distinguishing between Coordinator and Execution agents), session management with differentiated Quality of Service (QoS) guarantees, and dual routing mechanisms (ID-based and semantic-based resolution). The framework is intended to enable secure collaboration within a trust domain, providing a foundation for wide-area AI agent interconnection.

## What's New
The latest update (version 01, October 2025) introduces the **Agent Network Protocol (ANP)** as the foundational protocol for the framework. Key additions include a detailed **Abstract Protocol Flow** outlining the interaction between agents and registration/communication servers. It also formalizes the distinction between **Coordinator Agents** and **Execution Agents** and introduces two specific communication options: mediated through an Agent Communication Server or direct peer-to-peer. New sections on multimodal support and semantic-based route resolution have also been integrated to reflect evolving AI capabilities.
