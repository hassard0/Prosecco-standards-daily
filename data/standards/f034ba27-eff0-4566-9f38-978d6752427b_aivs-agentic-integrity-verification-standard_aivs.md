# AIVS: Agentic Integrity Verification Standard (AIVS)

**Status:** Draft | **Organization:** IETF

## Description
A portable, self-verifiable archive format for cryptographic proof of AI agent sessions. It uses SHA-256 hash-chained logs and Ed25519 signatures to ensure that agent actions are accounted for, unmodified, and verified offline.

## Latest Summary
The Agentic Integrity Verification Standard (AIVS) specifies a portable, self-verifiable archive format for providing cryptographic proof of AI agent sessions. The standard addresses a critical gap in AI observability: while existing platforms log agent actions, they typically lack cryptographic guarantees of completeness or authenticity. AIVS is designed to satisfy regulatory audit requirements, such as those found in the EU AI Act (Article 19) and ISO/IEC 42001, by providing a tamper-evident record of agent activities.

A core feature of the standard is the AIVS proof bundle, a gzip-compressed tar archive. This bundle contains a SHA-256 hash-chained audit log, an Ed25519 digital signature, a machine-readable manifest, and an embedded Python verification script. A unique "zero-dependency" design goal ensures that any party can verify a session's integrity using only the Python 3 standard library, without needing to contact external servers or blockchains.

The protocol supports two main formats:
1. **Full Proof Bundles**: Comprehensive archives for session-level audits.
2. **AIVS-Micro**: A lightweight, ~200-byte attestation containing six fields, optimized for continuous monitoring and high-frequency API contexts.

The standard defines a strict hash chain computation where each 'Audit Row' includes a reference to the previous row's hash (`prev_hash`). This structure ensures that any insertion, deletion, or reordering of agent actions is immediately detectable during verification. AIVS is intended to be domain-agnostic, applicable to any AI agent performing tasks like web navigation, data extraction, or transaction execution.

## What's New
The initial draft (**draft-stone-aivs-00**) introduces the first formal specification for cryptographically verifiable AI agent logs. Key highlights of this launch include:

*   **Hash-Chained Audit Logs**: Definition of a SHA-256 linear chain for agent actions.
*   **AIVS-Micro**: Introduction of a minimal 200-byte format for high-frequency scanning attestations.
*   **Self-Contained Verification**: Inclusion of an embedded `verify.py` script within bundles to allow for offline, dependency-free verification.
*   **Standard Alignment**: Mapping of AIVS components to existing frameworks like W3C Verifiable Credentials 2.0, IETF SCITT, and the EU AI Act.
*   **Identity Provisioning**: Use of Ed25519 signatures to link agent sessions to specific cryptographic identities.
