# DNS for AI Discovery (DNS-AID)

**Status:** Draft | **Organization:** IETF

## Description
A method for utilizing the Domain Name System (DNS) to facilitate scalable and interoperable discovery between AI agents. It defines a structured DNS namespace and record usage model to support metadata exchange and capability advertisement for AI entities.

## Latest Summary
DNS for AI Agent Discovery (DNS-AID) is a proposed IETF standard that utilizes the existing Domain Name System (DNS) infrastructure to enable scalable and interoperable discovery between AI agents. The specification defines a structured namespace convention and a usage model for existing DNS records—primarily Service Binding (SVCB), TXT, and TLSA records—within designated leaf zones, such as `_agents.example.com`. 

The primary goal of DNS-AID is to allow organizations to publish discovery metadata for their AI agents, enabling other agents to retrieve critical operational parameters (such as supported protocols, IP addresses, ports, and metadata like token pricing) prior to initiating a session. By leveraging established protocols like DNSSEC for integrity and DANE for authentication, the standard aims to provide a secure and automated "bootstrap" mechanism for agent-to-agent communication without requiring human intervention or modifications to core DNS protocol values. 

A key architectural advantage of DNS-AID is its non-disruptive, "opt-in" deployment model. It does not introduce new DNS opcodes or message formats, making it compatible with existing enterprise and cloud DNS tooling. The draft also incorporates Domain Control Validation (DCV) to allow agents to prove they are authorized to act on behalf of a specific domain. While the standard focuses on discovery and parameter exchange, it intentionally leaves the definition of internal agent information structures (like model cards or specific schemas) to separate, future specifications.

## What's New
The most significant recent update is the transition of the specification from its former name (BANDAID) to the current "DNS-AID" (DNS AI Agent Discovery) designation. The latest revision (01), published in March 2026, introduces more specific architectural examples, such as the use of `_index._agents.example.com` for service registries and `_a2a._agents.example.com` for agent-to-agent attributes. Additionally, the draft has been officially scheduled for discussion at the **IETF-125 DNSOP working group session**, specifically during the Thursday 08:30 morning block. This version also emphasizes integration with the IETF's ongoing work on Domain Verification Techniques to ensure authorized agent delegation.
