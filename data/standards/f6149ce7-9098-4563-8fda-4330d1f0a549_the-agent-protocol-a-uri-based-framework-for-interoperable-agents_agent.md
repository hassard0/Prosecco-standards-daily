# The agent:// Protocol -- A URI-Based Framework for Interoperable Agents (agent://)

**Status:** Draft | **Organization:** IETF

## Description
A URI template-based framework for addressing, invoking, and interoperating with autonomous and semi-autonomous software agents. It provides a layered architecture for capability discovery, contracts, and orchestration across different agent ecosystems.

## Latest Summary
The **agent:// protocol** is a proposed URI-based framework designed to provide a standardized, interoperable method for identifying and interacting with autonomous agents across different platforms and environments. Developed as an Internet-Draft within the IETF, the standard aims to solve the fragmentation in agent-to-agent and human-to-agent communication by defining a consistent addressing scheme.

The protocol structure follows the standard URI syntax (RFC 3986), utilizing the `agent:` scheme. It is designed to include components for identifying the agent's authority (host or platform), its specific identity (path), and optional parameters for capabilities or session-specific instructions through queries and fragments. By providing a uniform way to "address" an agent, the framework facilitates discovery, authentication, and the invocation of agentic functions regardless of the underlying implementation (e.g., Large Language Model backends, specialized robotic controllers, or software bots).

Key objectives of the framework include:
- **Interoperability:** Allowing agents from different developers or platforms to reference and communicate with each other.
- **Portability:** Enabling agent identities to remain consistent even if moved between hosting environments.
- **Extensibility:** Supporting various communication protocols (like WebSockets, HTTP, or decentralized protocols) through the resolution mechanism of the URI.

The standard is currently being developed through community feedback on GitHub and formal updates to the IETF Datatracker. Open questions often revolve around security considerations, such as how to prevent agent "spoofing" and how the URI integrates with existing Distributed Identifier (DID) standards.

## What's New
The most significant recent update is the release of **draft-narvaneni-agent-uri-02** in November 2024. This version includes refined definitions for the URI components and expanded sections on the security and privacy implications of globally addressable agents. The development team has also consolidated community discussions on the project's GitHub repository, focusing on creating a reference implementation and improving the documentation for the `agent://` resolution process. There is a growing focus on aligning the protocol with existing identification standards to ensure it does not duplicate efforts in the decentralized identity space.
