# Ontology-based Semantic Interaction for Internet of Agents (IOA-SI)

**Status:** Draft | **Organization:** IETF

## Description
Specifies a normative semantic layer for agent-to-agent interaction in the Internet of Agents (IoA), providing a common ontology model and JSON-LD serialization profile. It defines required classes and alignment rules for expressing capabilities, intents, and tasks to enable deterministic semantic interoperability.

## Latest Summary
This standard defines a normative semantic layer for agent-to-agent interaction within the Internet of Agents (IoA) ecosystem. Its primary goal is to resolve the semantic mismatches that occur when heterogeneous agents from different domains or vendors attempt to collaborate. By providing a structured meaning representation, the standard enables cross-domain discovery, reliable task decomposition, and predictable coordination.

The architecture places the semantic layer above interaction protocols and below application logic, ensuring it remains protocol-agnostic. It specifically focuses on expressing capabilities, intents, tasks, and contexts using a formal ontology. The specification mandates a core set of classes (e.g., `ioa:Agent`, `ioa:Capability`, `ioa:Intent`) and properties (e.g., `ioa:hasCapability`, `ioa:hasIntent`) to ensure baseline interoperability. 

A significant portion of the standard is dedicated to the **JSON-LD serialization profile**, which is the RECOMMENDED method for exchanging semantic data, though other RDF serializations are permitted. Furthermore, the document outlines procedures for **semantic negotiation and alignment**, allowing agents to reconcile different vocabularies through alignment artifacts and conflict resolution rules. While it provides the framework for deterministic semantic interoperability, it does not define the underlying transport, session, or security protocols.

## What's New
The latest update (version -01, February 2026) introduces a more robust **Ontology Model** incorporating RDF/OWL concepts for better cross-domain integration. Key additions include the formal definition of foundational classes like `ioa:SemanticProfile` and `ioa:Context`, as well as a more detailed **JSON-LD serialization profile** that includes requirements for `@context` and message semantic envelopes. The document also adds a specific subsection on **Semantic Negotiation and Alignment**, providing a clearer path for agents to handle heterogeneous ontologies through alignment artifacts and compatibility determination algorithms. Additionally, the draft now includes illustrative examples for NLIP HTTP encapsulation and cross-domain discovery flows.
