# AI Agent Authentication and Authorization (AI-AGENT-AUTH)

**Status:** Emerging | **Organization:** IETF

## Description
Proposes a model for authentication and authorization of AI agent interactions by leveraging WIMSE and OAuth 2.0. It aims to provide a framework for using existing standards and identifying gaps to guide future standardization for agents.

## Latest Summary
This informational document proposes a comprehensive model for the authentication and authorization of AI agents, treating them as autonomous workloads. Rather than inventing new protocols, the standard aims to consolidate existing "prior art" from the identity and security landscape to prevent fragmentation in the rapidly evolving AI field. The framework defines an AI Agent as a workload that iteratively interacts with Large Language Models (LLMs), tools, and services until a termination condition is met. 

The core of the proposal is the **Agent Identity Management System (AIMS)**, a conceptual model encompassing identity functions such as unique identifiers, cryptographic credentials, and attestation. The document advocates for leveraging established standards including **OAuth 2.0** for delegated authorization, **SPIFFE** for workload identity, and **WIMSE** (Workload Identity in Multi-System Environments) for architecture. It specifically addresses challenges such as agent-to-agent communication, human-in-the-loop requirements, and the need for dynamic "Transaction Tokens" to reduce risk during complex, multi-step agent operations. By providing this mapping of existing standards to AI-specific use cases, the draft seeks to identify functional gaps that may require future targeted standardization efforts.

## What's New
The standard is currently in its first iteration (**-00 draft**), introduced in March 2026. This initial release focuses on defining the AI Agent as a specific type of "workload" and establishes the **Agent Identity Management System (AIMS)** framework. Recent focus areas highlighted in the draft include the use of **WIMSE Proof Tokens (WPTs)** and **HTTP Message Signatures** for application-layer authentication, as well as exploring privacy considerations for cross-domain access and the discovery of Authorization Server capabilities in dynamic AI environments.
