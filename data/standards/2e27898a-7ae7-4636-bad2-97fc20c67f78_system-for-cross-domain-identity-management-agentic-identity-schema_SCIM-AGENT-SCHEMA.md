# System for Cross-domain Identity Management: Agentic Identity Schema (SCIM-AGENT-SCHEMA)

**Status:** Draft | **Organization:** IETF Project

## Description
Provides a platform-neutral schema for representing AI agents' identities in JSON format within the System for Cross-domain Identity Management (SCIM). It enables agentic identities to be transferred, authenticated, and authorized to interact with services.

## Latest Summary
The **System for Cross-domain Identity Management: Agentic Identity Schema** is an IETF Internet-Draft that extends the base SCIM specifications (RFC 7644 and RFC 7643) to accommodate the unique identity requirements of AI agents. As AI agents increasingly interact with cloud services, there is a growing need to provision, manage, and authorize these non-human entities using standardized protocols. 

The core of this standard is the introduction of the `AgenticIdentity` resource type (`urn:ietf:params:scim:schemas:core:2.0:AgenticIdentity`). Unlike traditional human user schemas, this schema focuses on attributes relevant to automated entities, such as `agenticApplicationId` for correlating multiple identities to a single application and `oAuthClientIdentifiers` to facilitate authentication via OAuth token exchange (RFC 8693). It also maintains parity with standard SCIM lifecycle controls, allowing agents to be assigned to groups, roles, and entitlements. 

Key technical decisions in the draft include:
*   **Schema Extension:** Augmenting the standard SCIM "members" attribute to explicitly allow references to `AgenticIdentity` resources.
*   **Ownership:** The inclusion of an `owners` attribute to map agents to responsible parties (users or groups).
*   **Interoperability:** Alignment with existing OAuth claims (e.g., `aud`, `iss`, `sub`) within the agent's identity record.

The specification is currently in the "Informational" track and is being discussed within the SCIM Working Group. Open areas typically include refined security considerations regarding agent impersonation and the specific mechanics of delegating access rights from human owners to automated agents.

## What's New
The most significant update in the **-01 iteration (August 2025)** is the refinement of the `AgenticIdentity` resource schema. Key changes include the formalization of the `agenticApplicationId` for multi-identity correlation and the detailed structure of the `oAuthClientIdentifiers` complex attribute. This version specifically addresses how agents authenticate via **OAuth token exchange**, mapping sub-attributes like `issuer`, `subject`, and `audiences` directly to standard JWT claims. Furthermore, the draft now explicitly defines updates to the existing SCIM Group schema, allowing the `members` attribute to natively support the `AgenticIdentity` type, ensuring that AI agents can be managed within standard enterprise group structures.
