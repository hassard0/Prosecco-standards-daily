# The Intent Token: A Cryptographic Authorization Primitive for Autonomous Agents (Intent Token)

**Status:** Emerging | **Organization:** IETF Outer Space

## Description
A cryptographic authorization primitive that binds autonomous agent actions to human-declared authorization envelopes. It fills the gap in existing frameworks by governing what an autonomous agent is authorized to do at the moment of action.

## Latest Summary
The **Intent Token** is a proposed cryptographic authorization primitive designed specifically for autonomous AI agents. While current standards like OAuth 2.0 and OpenID Connect (OIDC) manage identity (the "Who") and session-level access (the "What resources"), the Intent Token aims to bridge the "authorization gap" by defining what an agent is specifically authorized to **do** at the moment of action. 

The standard introduces a signed, time-bounded JSON Web Token (JWT) structure that binds an agent's intended action to a human-declared authorization envelope. This mechanism is intended to prevent "intent blindness"—where an agent interprets its scope in unintended ways—and addresses the "confused deputy" problem where agents might be manipulated by adversarial prompts. Key features include:
*   **Action Binding:** Verification of specific actions (e.g., financial limits, permitted instruments) before execution.
*   **Temporal Sharding:** Tokens are bound to short-lived "shards" (default 512 seconds), requiring frequent re-validation.
*   **Delegation Chains:** A cryptographic mechanism to ensure sub-agents cannot exceed the authorization scope of the primary agent.
*   **SNAP-BACK:** A mandatory enforcement rule that cancels any action failing validation before it has physical or market effects.

The Intent Token is part of the broader **Intent Bound Authorization (IBA)** framework and is designed to be model-agnostic and transport-agnostic, allowing it to compose with existing OAuth infrastructure.

## What's New
The standard was recently introduced to the IETF as an **Informational Internet-Draft (version 00)** on March 19, 2026. This initial version establishes the core JWT structure for the Intent Token, including required claims like `ibt_ver`, `principal`, `declared_intent`, and `shard`. It also introduces the concept of **SNAP-BACK enforcement**, requiring that any authorization failure results in the immediate cancellation of an agent's action before it takes effect. The draft explicitly positions itself alongside Google DeepMind's "Delegation Capability Tokens" (DCTs) as a potential standard to fill the global gap in autonomous agent governance.
