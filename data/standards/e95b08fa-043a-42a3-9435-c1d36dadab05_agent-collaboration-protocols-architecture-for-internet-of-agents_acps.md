# Agent Collaboration Protocols Architecture for Internet of Agents (ACPs)

**Status:** Draft | **Organization:** IETF

## Description
An architecture for the Internet of Agents (IoA) that facilitates interconnection and collaboration among heterogeneous agents. It covers agent registration, identity authentication, discovery, interaction, tool invocation, and monitoring.

## Latest Summary
The **Agent Collaboration Protocols (ACPs) Architecture** is a proposed framework designed to facilitate the "Internet of Agents" (IoA). Its primary objective is to enable seamless interconnection and collaboration among heterogeneous AI agents, moving beyond single-agent limitations and proprietary vendor constraints. The architecture is defined by a multi-centralized structure composed of autonomous domains, ensuring scalability and reliability for large-scale agent networks.

The specification organizes the IoA environment into five conceptual domains: **User, Agent, Identity Management, Interconnection Service, and Resource Access**. Key functional components within these domains handle tasks such as trusted registration, identity authentication via Agent Identity Codes (AIC), and capability discovery through Agent Capability Specifications (ACS). The protocol supports both point-to-point and group interaction modes, as well as tool invocation and real-time monitoring.

By providing standardized interfaces and communication mechanisms, ACPs aim to build the foundational infrastructure for agent-to-agent negotiation, task allocation, and secure data exchange in a cross-domain environment.

## What's New
The latest update (**draft-liu-dmsc-acps-arc-03**), released in January 2026, solidifies the functional architecture of the Agent Autonomous Domain. Key updates include a detailed breakdown of the five conceptual domains and the introduction of core functional elements like the **Agent Identity Code (AIC)** and **Agent Capability Specification (ACS)**. This version specifically addresses multi-centralized management, allowing for autonomous domains to maintain their own registration and discovery functions while supporting cross-domain interoperability. It also refines the operational scenarios for agent onboarding, partner selection, and tool invocation.
