# Agent Identity Protocol: Agentic Authentication and Authorized Policy Enforcement (AIP)

**Status:** Draft | **Organization:** IETF

## Description
An open standard for verifiable identity and policy enforcement for AI agents, providing unique identifiers and cryptographic signing for agent actions. It utilizes a two-layer architecture comprising an identity registry and an enforcement proxy to ensure secure, authorized tool execution.

## Latest Summary
The Agent Identity Protocol (AIP) is an IETF Internet-Draft proposed to establish a standardized framework for the identity and authorization of AI agents. The protocol addresses the "identity gap" where AI agents currently operate using human credentials, making it impossible for downstream services to distinguish between human and automated actions. This lack of distinction leads to security vulnerabilities, auditability challenges, and compliance issues.

The AIP is architected into two distinct but cooperative layers:
*   **Layer 1 (Identity):** Establishes a verifiable identity for every agent. Agents are registered with an AIP Registry, which assigns a unique Agent ID and stores the agent's public key. Agents must then sign every outbound tool call with their private key, creating a "AIP Token" (Agent Attestation Token) for verification.
*   **Layer 2 (Enforcement):** Utilizes an AIP Proxy that sits between the AI client and tool servers. This proxy intercepts calls, verifies signatures against the registry, and evaluates requests against a declarative **AgentPolicy**. This policy can trigger outcomes such as allowing the call, denying it, or putting it on hold for "Human-in-the-Loop" (HITL) approval.

The protocol is designed to be transport-agnostic but identifies the Model Context Protocol (MCP) as its primary initial target for tool-call interfaces. It does not replace existing authentication methods like OAuth or mTLS but adds a necessary layer of agent-specific attribution and policy control.

## What's New
The standard is currently in its inaugural phase with the release of **draft-aip-agent-identity-protocol-00** on March 16, 2026. Key newly defined components include the **AIP Token** for agent attestation, the specification for an **AIP Registry** to manage public keys, and the introduction of a YAML-based **AgentPolicy** schema. The protocol specifically introduces "Human-in-the-Loop" (HITL) and Data Loss Prevention (DLP) rules as core components of the enforcement layer. activity is currently focused on the initial solicitation of feedback from the IETF community.
