# Entity Attestation Token (EAT) Profile for Autonomous AI Agents (EAT-AI)

**Status:** Draft | **Organization:** IETF

## Description
A profile for the Entity Attestation Token (EAT) to support remote attestation of autonomous AI agents. It specifies standardized claims for attesting AI model integrity, data provenance, and inference-time access policies using CBOR or JSON Web Tokens.

## Latest Summary
The **Entity Attestation Token (EAT) Profile for Autonomous AI Agents** is an informational IETF Internet-Draft that defines a specialized profile of the Entity Attestation Token (RFC 9711). Its primary objective is to enable remote attestation of autonomous AI agents across different domains, ensuring that verifiers can confirm an agent's technical and operational integrity before granting access to resources or data.

The draft addresses a critical security gap in AI deployment: the lack of standardized claims to verify AI-specific artifacts. By leveraging the IETF Remote ATtestation procedureS (RATS) architecture, this profile allows verifiers to assess an agent based on its model integrity (weights and architecture), training provenance (dataset origin and privacy properties), and runtime authorization (allowed APIs and capabilities). This is particularly relevant for autonomous systems where minimal human oversight increases the risk of model tampering or non-compliant data usage.

Key technical components include:
*   **Core Claims**: Standardized fields for both CBOR Web Tokens (CWT) and JSON Web Tokens (JWT) covering model IDs, cryptographic digests of weights/architecture, and references to AI Software Bill of Materials (AI-SBOM).
*   **Privacy Features**: Support for claims like `dp-epsilon` to verify differential privacy levels used during training.
*   **Interoperability**: Includes optional extensions for 5G/6G network functions to align with ETSI ENI and 3GPP architectures, such as network slice-type authorization.
*   **Layered Trust**: Guidelines for handling multi-agent platforms and multi-model agents through composite attestation.

The profile is designed to complement existing software supply chain standards (like SBOMs) rather than replace them, focusing specifically on the verifiable evidence needed for real-time trust decisions.

## What's New
The latest update, **version -01 (February 2026)**, further refines the claim definitions for AI agents. Key additions include the **`ai-sbom-ref`** claim, which provides a link to Software Bill of Materials specifically for AI runtime dependencies. The update also clarifies the use of **cryptographic digests** for model weights and computational graphs, moving from bare byte strings to a structured format to ensure algorithm agility. Furthermore, it expanded on **multi-component attestation**, providing guidance for complex environments like multi-agent platforms where trust must be layered across various software and hardware components.
