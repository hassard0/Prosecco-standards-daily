# AGENTS.md (AGENTS.md)

**Status:** Approved | **Organization:** Factory

## Description
AGENTS.md is a simple, open companion to README.md that gives coding agents a predictable, machine-readable place for project guidance (setup commands, test workflows, code style, and PR conventions) reducing README noise. Adopted by 20k+ GitHub repos and portable across tools like Cursor, Aider, RooCode, Zed, and more.

## Latest Summary
**AGENTS.md** is an emerging community standard designed to provide persistent, project-specific instructions and context to AI coding agents. Similar to established files like `.cursorrules` or `CLAUDE.md`, an `AGENTS.md` file resides in a project's root directory and contains guidelines on coding conventions, architectural patterns, and preferred libraries. This ensures that any AI tool interacting with the codebase—regardless of the specific vendor or IDE—adheres to the same project-level constraints and high-level goals.

The standard aims to solve the "fragmentation" problem in AI agent configuration. By using a vendor-neutral filename, developers avoid maintaining multiple redundant configuration files (such as `.cursorrules`, `.windsurfrules`, and `CLAUDE.md`). The standard encourages a structured Markdown format that typically includes sections for project overviews, technology stacks, specific coding standards, and "always-load" instructions.

Key technical decisions revolve around interoperability. Leading AI coding tools and IDEs are increasingly adopting support for `AGENTS.md` by automatically indexing its content during the initial prompt construction or "agentic" reasoning phases. This allows the AI to remain "in character" for the project without the user needing to manually repeat instructions in every chat session. Open questions remain regarding the hierarchical merging of instructions (e.g., global user rules vs. project-specific `AGENTS.md` rules) and how to handle conflicts between different instruction files if multiple exist in the same repository.

## What's New
The standard has gained rapid momentum as a vendor-neutral alternative to editor-specific configuration files. Recent updates across the ecosystem include:
- **Zed Editor Integration:** Zed now explicitly lists `AGENTS.md` in its "Rules" documentation as a supported file for project-level AI instructions.
- **Aider "Always Load" Support:** Aider has standardized on checking for `AGENTS.md` to automatically apply coding conventions to every session.
- **Goose Extension:** The Goose agent platform has integrated `AGENTS.md` to provide project-specific context to its underlying LLM workflows.
- **Community Adoption:** There is a growing movement in GitHub repositories to replace `.cursorrules` with `AGENTS.md` to ensure compatibility across Cursor, Windsurf, Zed, and CLI-based agents.
