# Agent Directory Service (ADS)

**Status:** Draft | **Organization:** IETF

## Description
The Agent Directory Service (ADS) is a distributed directory service designed to store metadata for AI agent applications. It enables the discovery of agents with specific skills through directory records and an interconnected content-routing protocol.

## Latest Summary
The Agent Directory Service (ADS) is a distributed directory infrastructure designed specifically for the AI agent ecosystem. It provides a standardized mechanism to store, discover, and verify metadata for AI agent applications (Multi-Agent Systems or MAS). The service addresses the complexities of agent discovery by allowing developers to search for agents based on specific skills, performance characteristics (cost, latency), and resource requirements. 

Architecturally, ADS is built on decentralized storage principles. it utilizes **Open Container Initiative (OCI)** specifications to package agent records as artifacts, allowing the standard to leverage existing container registry tools, security scanners, and authentication mechanisms. A core component of the system is the use of **Content Identifiers (CIDs)**, which ensure data immutability, deduplication, and location independence. 

For discovery, ADS employs a two-level mapping architecture using **Distributed Hash Tables (DHT)** and content-routing protocols. This enables efficient matching of agent capabilities against hierarchical skill taxonomies (using the **Open Agent Schema Framework - OASF**). The standard also emphasizes security through cryptographic integrity, digital signatures for content provenance, and trust boundaries to mitigate threats in decentralized networks. By using OASF, ADS can handle both objective metrics like GPU requirements and subjective evaluations like user ratings.

## What's New
The latest update (version **-01**, released February 2024) significantly refines the **Storage Architecture** and **Discovery Process**. Key additions include:

*   **OCI Integration:** Formalized the use of OCI Image Manifests for storing agent records as "Record Artifacts," ensuring compatibility with the broader container ecosystem.
*   **Two-Level Mapping Architecture:** Detailed the DHT-based discovery process, specifically how skills map to Content Identifiers (CIDs) and CIDs map to PeerIDs.
*   **Skill Taxonomy:** Introduced hierarchical skill taxonomies for search optimization, allowing for multi-dimensional searches across domains and modules.
*   **Security Model Enhancements:** Added comprehensive sections on cryptographic integrity, digital signatures for provenance, and threat mitigation strategies within decentralized networks.
*   **Open Agent Schema Framework (OASF):** Deepened the integration with OASF for structured metadata modeling.
