# Using LISP as a Network Substrate for AI Agent Communication (LISP-AI-Agent)

**Status:** Emerging | **Organization:** IETF

## Description
Explores how the Locator/ID Separation Protocol (LISP) can serve as a network substrate for distributed AI agents across cloud, edge, and endpoint environments. It outlines extensions to LISP to support context-aware mapping and AI agent-centric communication.

## Latest Summary
This standard proposes the use of the Locator/ID Separation Protocol (LISP) as a foundational network layer to facilitate communication between distributed AI agents. As AI systems become increasingly decentralized across cloud, edge, and endpoint environments, they require networking capabilities that traditional IP cannot easily provide, such as location transparency, persistent identity during mobility, and logical isolation. 

The draft, authored by Wei Wang and Chongfeng Xie of China Telecom, argues that LISP's inherent architecture—which decouples an endpoint's identity (EID) from its routing location (RLOC)—naturally addresses these needs. By assigning each AI agent a stable EID, the agent can migrate across different physical hosts or networks without disrupting active communication sessions. Furthermore, LISP's Instance ID (IID) capability allows for the creation of isolated "Agent Groups," ensuring security and multi-tenancy even on shared physical infrastructure.

Key technical contributions of the document include proposed extensions to the LISP framework to make it "context-aware." This involves enhancing the LISP Mapping System to store and retrieve specific AI agent attributes, such as processing latency (Latency_SLA) and hardware capability tags (e.g., GPU availability). These extensions would allow Ingress Tunnel Routers (ITRs) to make intelligent, policy-based routing decisions, ensuring agents are connected to hosts that meet their specific computational or performance requirements. Additionally, the draft suggests enhanced Map-Request/Map-Reply semantics and more efficient metadata update mechanisms to support high-frequency state changes in dynamic AI environments.

## What's New
The standard is currently in its very first iteration (**version 00**), introduced at the end of February 2025. The most significant "new" development is the formal introduction of **Context-Aware Mapping** specifically for AI workloads. This includes the proposal for the LISP Mapping System to carry non-topological metadata like **Processing Latency (Latency_SLA)** and **Hardware Capability Tags** (e.g., "I need a GPU"). It also introduces the concept of **Dynamic Metadata Updates**, allowing Edge Tunnel Routers (ETRs) to update agent status (load/intent) without the overhead of full de-registration, addressing the high-mobility and high-state-change nature of autonomous AI agents.
