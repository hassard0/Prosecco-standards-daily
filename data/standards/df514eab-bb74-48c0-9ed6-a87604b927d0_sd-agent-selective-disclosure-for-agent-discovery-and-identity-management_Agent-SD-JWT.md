# SD Agent: Selective Disclosure for Agent Discovery and Identity Management (Agent-SD-JWT)

**Status:** Draft | **Organization:** IETF

## Description
Integrates Selective Disclosure for JWTs (SD-JWT) with Agent-to-Agent (A2A) systems to provide privacy-preserving agent discovery and identity management. It specifies the SD-Card format to enable selective disclosure of agent capabilities, contact information, and metadata.

## Latest Summary
The **SD Agent** standard defines a framework for integrating Selective Disclosure for JSON Web Tokens (SD-JWT) with Agent-to-Agent (A2A) systems. Its primary goal is to provide a privacy-preserving mechanism for autonomous agent discovery and identity management. By utilizing the **SD-Card** format—an SD-JWT encoding of traditional Agent Cards—the protocol allows agents to reveal only the specific capabilities, contact information, and operational metadata required for a given interaction context.

Key architectural components include **Agent Registries** (Issuers), **Agents** (Holders), and **Services/Clients** (Verifiers). The standard addresses critical limitations in legacy A2A implementations, such as information leakage, weak authentication, and context-blind disclosure. By incorporating **Key Binding**, it ensures that the entity presenting the SD-Card is the legitimate holder of the associated private key, thereby preventing replay attacks and ensuring cryptographic integrity.

The specification also introduces **Discovery Contexts** (e.g., public vs. internal), which dictate which claims are revealed during the discovery phase. This prevents mass enumeration of agent capabilities by unauthorized parties and supports unlinkable presentations to avoid cross-context tracking of agent activities. Furthermore, it details integration paths with the Model Context Protocol and advanced authorization flows like role-based disclosure and sub-agent delegation.

## What's New
The latest update (version 02) introduces the **SD-Card** format, specifically tailored for the Agent-to-Agent (A2A) ecosystem. Key additions include:
*   **Selective Disclosure Claims:** Specific definitions for hiding/revealing agent skills, capabilities, security schemes, and provider information.
*   **Key Binding Support:** Integration of the `cnf` (confirmation) claim to cryptographically bind an agent's identity to a public/private key pair.
*   **Discovery Protocols:** New guidance on privacy-protected discovery via Well-Known URIs and Registry-based lookups.
*   **System Architecture:** A formalized four-party model (Registry, Agent, Service, Client) for managing disclosable credentials.
*   **IANA Registrations:** Initial requests for media types (e.g., `application/vnd.sd-jwt+json`) and Verifiable Credential types.
