# Agent Authorization Profile (AAP) for OAuth 2.0 (AAP)

**Status:** Draft | **Organization:** IETF Shattered (Individual) | **⚠️ Expired**

## Description
An authorization profile for OAuth 2.0 and JWT designed for autonomous AI agents. It extends existing standards with structured claims for agent identity, task context, operational constraints, and human oversight.

## Latest Summary
The Agent Authorization Profile (AAP) for OAuth 2.0 is a specialized authorization profile designed specifically for autonomous AI agents. Built upon existing standards like OAuth 2.0 (RFC 6749) and JSON Web Tokens (JWT, RFC 7519), AAP addresses the unique security challenges posed by AI agents, such as high-frequency autonomous actions, the absence of real-time human approval, and the need for complex delegation across multiple tools. 

Key technical contributions of the profile include:
- **Structured Claims:** Introduction of new JWT claims to represent agent identity (`agt`), task context (`tsk`), and operational constraints (`cns`).
- **Capability-Based Authorization:** Moving beyond simple scopes to "capabilities" that include fine-grained action permissions and lifecycle constraints.
- **Delegation Management:** Defining an auditable delegation chain (`dlg`) that tracks the history of token exchanges and ensures privilege reduction at each level.
- **Human Oversight:** Mechanisms to encode requirements for human intervention or approval within the token itself.
- **Proof-of-Possession:** Strict requirements for binding tokens to the agent's cryptographic identity to prevent impersonation and token theft.

The profile is intended for Standards Track status and focuses on machine-to-machine (M2M) scenarios where context-aware, auditable authorization is critical. It explicitly avoids defining internal AI model behaviors or ethical decision-making, focusing instead on the security handshake between agents, Authorization Servers, and Resource Servers.

## What's New
The latest update (version -01) introduces several critical technical refinements to the Agent Authorization Profile. Key additions include a detailed **Action Name Grammar (ABNF)** for standardizing how capabilities are requested, and a comprehensive **JSON Schema** in the appendix for validating AAP tokens. The update significantly expands the **Security and Privacy Considerations** sections, offering specific guidance on data minimization, cross-domain correlation, and cryptographic key management. Furthermore, version -01 provides elaborated **Token Exchange** examples that demonstrate "Privilege Reduction," showing how an agent can exchange a broad token for a more restricted, tool-specific one while maintaining an auditable delegation chain. New error codes specifically for AAP-related failures (e.g., `insufficient_oversight`, `delegation_depth_exceeded`) have also been registered.
