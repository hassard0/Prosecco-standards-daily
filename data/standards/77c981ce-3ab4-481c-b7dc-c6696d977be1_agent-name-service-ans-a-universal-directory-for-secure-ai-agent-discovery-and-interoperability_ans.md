# Agent Name Service (ANS): A Universal Directory for Secure AI Agent Discovery and Interoperability (ANS)

**Status:** Draft | **Organization:** IETF

## Description
Agent Name Service (ANS) is a DNS-inspired architecture and universal directory protocol designed for secure AI agent discovery. It provides a protocol-agnostic registry using PKI certificates to verify agent identity and supports multiple communication standards like A2A, MCP, and ACP.

## Latest Summary
The Agent Name Service (ANS) is a proposed universal directory architecture designed to address the critical lack of a public framework for discovering and verifying AI agents. As agent-to-agent (A2A) traffic is projected to become a major portion of internet data, ANS provides a protocol-agnostic registry that bridges emerging standards like Google's A2A, the Model Context Protocol (MCP), and the Agent Communication Protocol (ACP). 

The framework is heavily inspired by the Domain Name System (DNS) but enhances it with semantic and security features necessary for autonomous agents. At its core, ANS leverages Public Key Infrastructure (PKI) for verifiable identity, using X.509 digital certificates to establish a root of trust through Certificate Authorities (CA) and Registration Authorities (RA). This ensures that agent discovery isn't just about finding a network address, but about verifying the agent's identity and its specific capabilities.

Key architectural innovations include a **Protocol Adapter Layer**, which translates between the registry's internal data and various external protocol formats, and a **Formal Naming Structure** (e.g., `protocol://agentID.capability.provider.version`) that allows for granular, capability-aware resolution. The specification also introduces a structured request/response schema based on JSON Schema to handle lifecycle events like registration, renewal, and revocation. 

Security is a primary focus of the standard, featuring a comprehensive threat analysis based on the MAESTRO framework. It addresses risks such as agent impersonation, registry poisoning, and Man-in-the-Middle (MitM) attacks through mandatory PKI integration and secure resolution algorithms. By providing this foundational infrastructure layer, ANS aims to enable a scalable, trustworthy, and interoperable ecosystem for the next generation of AI agents.

## What's New
The Agent Name Service (ANS) has been introduced via its first Internet-Draft (`draft-narajala-ans-00`) in May 2025. This initial release establishes a foundational architecture for AI agent discovery, featuring a modular **Protocol Adapter Layer** to support MCP, ACP, and A2A protocols. Key recent developments include the formalization of a DNS-like naming convention (e.g., `protocol://agentID.capability.provider.version`) and the integration of PKI-based lifecycle management. The draft also introduces a **MAESTRO-based threat analysis** to secure agent registries against impersonation and poisoning, and provides a structured JSON Schema for protocol-agnostic communication between agents and the directory service.
