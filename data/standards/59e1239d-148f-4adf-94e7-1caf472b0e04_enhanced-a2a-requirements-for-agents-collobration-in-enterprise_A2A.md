# Enhanced A2A Requirements for Agents Collobration in Enterprise (A2A)

**Status:** Draft | **Organization:** IETF

## Description
This document proposes enhanced requirements for the Agent-to-Agent (A2A) protocol specifically tailored for enterprise applications. It focuses on facilitating collaboration between autonomous agents within an enterprise environment.

## Latest Summary
The "Enhanced A2A Requirements for Agents Collaboration in Enterprise" is an informational Internet-Draft aimed at extending the Agent2Agent (A2A) protocol to meet the specific demands of enterprise environments. While the original A2A protocol (launched by Google Cloud) provides a foundation for AI agents to collaborate, this proposal identifies critical gaps in centralized discovery, complex business process management, and stringent security requirements prevalent in large-scale organizations.

Key technical requirements introduced in the draft include:
* **Enhanced Discovery:** Support for centralized directory services to handle thousands of concurrent agents, reducing search latency and enabling intent-based matching that maps high-level user goals to multi-agent chains.
* **Complex Collaboration:** Mechanisms for creating temporary collaboration groups that automatically disband upon task completion, as well as formal conflict resolution rules for handling data inconsistencies between agents.
* **Enterprise Security:** Fine-grained permission controls mapped to data sensitivity levels and task types, alongside mandatory collaboration behavior audit logs to ensure compliance and traceability.

The standard is currently in the early draft stage within the IETF "dispatch" group, intended to provide a unified standard for how specialized AI agents (e.g., finance, R&D, and HR agents) interact within a secure corporate perimeter.

## What's New
The latest update, version **-01** (released November 28, 2025), focuses on formalized requirements for **centralized directory services** and **temporary collaboration groups**. It specifically addresses the scalability challenges of managing hundreds to thousands of specialized agents within a single enterprise. Recent additions emphasize the need for "intent-based discovery," which allows systems to map complex user prompts to specific combinations of agents. Furthermore, the draft now explicitly calls for **Task Progress Reporting**, enabling remote agents to report execution percentages to maintain visibility over long-running business processes.
