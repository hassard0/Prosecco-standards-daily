# VCAP: Verified Commerce for Agent Protocols (VCAP)

**Status:** Emerging | **Organization:** IETF

## Description
An open standard for settling financial transactions between autonomous AI agents using cryptographically verifiable proof of work delivery. It defines message formats and state machines for holding funds in escrow and releasing payments based on machine-verifiable evidence.

## Latest Summary
Verified Commerce for Agent Protocols (VCAP) is a proposed open standard designed to facilitate secure financial transactions between autonomous AI agents. As AI agents increasingly operate independently—hiring other agents and delegating subtasks—a need has emerged for a vendor-neutral settlement layer that ensures payment is only released upon delivery of verified work. VCAP addresses this by providing a framework of message formats, state machines, and cryptographic bindings that complement existing agent communication protocols like Google A2A or the Agent Protocol.

The core of VCAP is its verification-agnostic approach, allowing for various verification engines—ranging from browser automation and LLM evaluation to human review—to produce compliant proofs. The protocol operates through seven distinct phases: Negotiation, Escrow, Execution, Delivery, Verification, Settlement, and Escalation. This structured approach ensures that funds are held in escrow by a marketplace or platform and are only released to the provider agent once an independent verifier confirms the work meets the agreed-upon criteria.

Technically, VCAP relies on JSON-based message formats for negotiation, escrow holds, and verification callbacks. It emphasizes cryptographic auditability, where every settlement is tied to a proof hash and signature. This allows for independent verification of the transaction history and the evidence of work. Furthermore, the protocol is designed for graceful degradation; if automated verification proves ambiguous or encounters a timeout, the system is designed to route the dispute to human review. Use cases for VCAP include any scenario where machine-to-machine commerce requires trustless settlement, such as automated content generation, data extraction, or specialized API services performed by one AI agent for another.

## What's New
The VCAP specification was recently introduced as an **Internet-Draft (version 00)** in March 2026. This initial release establishes the foundational architecture for AI agent commerce, including:
*   **A Seven-Phase Transaction Flow:** Defining the lifecycle from negotiation and escrow to cryptographic settlement.
*   **Verification Engine Interoperability:** Support for diverse verification methods (LLM-eval, browser automation, etc.) through standardized "Verification Hints."
*   **Cryptographic Bindings:** The introduction of proof hashes and signatures to ensure that settlement is machine-verifiable and auditable.
*   **Marketplace Integration:** Designed to be rail-agnostic, the protocol supports settlement across traditional payment systems (like Stripe), internal wallets, or cryptocurrency.
*   **Compatibility:** Explicit alignment with existing agent standards like Google A2A and the Agent Protocol to act as a specialized settlement layer.
