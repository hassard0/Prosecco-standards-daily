# Agent Identity Protocol (AIP): Verifiable Delegation for AI Agent Systems (AIP)

**Status:** Draft | **Organization:** IETF

## Description
The Agent Identity Protocol (AIP) provides a framework for verifiable and delegable identity for AI agent systems using Invocation-Bound Capability Tokens (IBCTs). It supports both single-hop JWT-based interactions and multi-hop delegation chains using Biscuit tokens with Datalog policies.

## Latest Summary
The Agent Identity Protocol (AIP) is a proposed standard designed to provide a verifiable and delegable identity framework for AI agent systems. It specifically addresses critical security gaps in emerging AI infrastructures, such as the Model Context Protocol (MCP) and Agent-to-Agent Protocol (A2A), which often lack robust built-in authentication or fail to track multi-hop delegation. AIP introduces **Invocation-Bound Capability Tokens (IBCTs)**, which act as cryptographic artifacts combining identity, authorization, scope constraints, and provenance.

The protocol defines two primary token modes:
1.  **Compact Mode (JWT):** Utilizes JSON Web Tokens with Ed25519 signatures for efficient, single-hop interactions.
2.  **Chained Mode (Biscuit):** Leverages Biscuit tokens with append-only blocks and Datalog policy evaluation to support complex, multi-hop delegation chains where each subsequent "hop" can cryptographically attenuate (narrow) its permissions.

AIP provides two identity schemes: DNS-based identifiers (`aip:web:`) for stable, long-lived agents and self-certifying identifiers (`aip:key:`) based on public keys for ephemeral agents. A core requirement of the protocol is **Scope Attenuation**, ensuring that at each step of delegation, permissions (tools, budget, domains, and time) can only stay the same or decrease, never expand. The specification includes formal protocol bindings for MCP, A2A, and generic HTTP APIs, defining how tokens should be transported and verified across these different communication layers.

## What's New
The Agent Identity Protocol (AIP) was recently introduced (March 2026) to solve the identity problem in multi-agent architectures. Recent updates highlight a survey of roughly 2,000 MCP servers that found a total lack of authentication, providing the primary motivation for this draft. New features include **Invocation-Bound Capability Tokens (IBCTs)** that support both single-hop (JWT) and multi-hop (Biscuit) delegation. The draft also introduces **Budget Semantics**, allowing delegators to set USD spending ceilings on a per-token basis, and **Completion Blocks** for recording execution outcomes and resource usage. Additionally, new protocol bindings have been defined for the **Model Context Protocol (MCP)** and **Agent-to-Agent (A2A)** interactions.
