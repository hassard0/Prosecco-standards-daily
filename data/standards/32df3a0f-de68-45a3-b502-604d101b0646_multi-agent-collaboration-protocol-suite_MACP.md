# Multi-agent Collaboration Protocol Suite (MACP)

**Status:** Draft | **Organization:** IETF

## Description
A protocol suite that enables scalable, secure, and semantically driven collaboration among distributed agents across heterogeneous networks. It introduces Agent Gateways for registration, authentication, and capability management while preserving direct peer-to-peer semantic interactions.

## Latest Summary
The **Multi-agent Collaboration Protocol Suite (MACP)** is an IETF Internet-Draft aimed at establishing a standardized framework for scalable, secure, and semantically driven collaboration between autonomous software agents across heterogeneous networks. As multi-agent systems grow in complexity and cross administrative boundaries, traditional message-based models often fail to capture agent-level capabilities or dynamic task decomposition. MACP addresses this by introducing a structured architecture that separates the control and forwarding planes while ensuring least-privilege authorization.

Key functional entities in the MACP architecture include:
*   **Agents:** Autonomous software entities that perform tasks, maintain their own Agent Identity Codes (AIC), and engage in peer-to-peer semantic interactions.
*   **Agent Gateway:** A flexible infrastructure entity used for agent registration, capability management, and message distribution. It acts as a mediator for control and policy enforcement while allowing direct agent-to-agent communication.
*   **Agent Management Center (AMC):** The trust anchor responsible for the lifecycle management of agent identities and the issuance of credentials (e.g., X.509 certificates).

The protocol suite is composed of several specialized protocols: **ARP** for registration, **AAAP** for authentication and authorization, **CDSP** for synchronizing capability digests across gateways, and **TIP** for task-based orchestration. By defining these roles and protocols, MACP enables agents to discover each other's capabilities and collaborate on complex tasks through a vendor-neutral, interoperable interface. The specification is currently being developed within the **DMSC (Distributed Multi-agent System Collaboration)** Working Group context.

## What's New
The latest update, **version 02 (February 2026)**, provides significant refinements to the Multi-agent Collaboration Function Entity Architecture. It introduces a more detailed breakdown of the **Agent Gateway**'s role as a flexible infrastructure component that can range from a network service to a dedicated hardware gateway. The update also clarifies the **Agent Identity Code (AIC)** and **Agent Capability Specification (ACS)** as critical data objects for global uniqueness and service discovery. Additionally, new deployment examples have been added, specifically targeting **Fixed Networks** and **Mobile Operator Networks**, to demonstrate how the protocol suite handles agent collaboration within real-world telecommunications infrastructures.
