# Agent Payments Protocol (AP2)

**Status:** Draft | **Organization:** Google

## Description
The Agent Payments Protocol (AP2) facilitates secure, decentralized, and privacy-protecting payments by generative AI agents on behalf of users. It uses Verifiable Digital Credentials (VDCs) to enable interoperability within agentic ecosystems alongside protocols like A2A and MCP.

## Latest Summary
The Agent Payments Protocol (AP2) is a decentralized solution developed by Google to enable generative AI agents to execute financial transactions on behalf of users securely and privately. The protocol addresses the unique challenges of agentic commerce, where autonomous systems must interact with payment rails while maintaining strict user authorization and privacy standards. AP2 is designed as an interoperable standard that fits into a broader ecosystem of agentic protocols, including the Agent-to-Agent (A2A) and Model Context Protocol (MCP).

Key components of the AP2 architecture include a specification for secure communication, extensions specifically for A2A interactions, and core concepts that define the lifecycle of an agentic transaction. By providing a standardized way for agents to handle payment credentials and transaction approvals, AP2 aims to reduce friction in the global nature of automated commerce while ensuring that users remain in control of their financial data.

## What's New
The Agent Payments Protocol (AP2) has recently formalized its alignment with other emerging agentic standards, specifically releasing an **A2A (Agent-to-Agent) Extension**. This extension allows agents to negotiate payment terms and hand off transaction details across different autonomous platforms. Additionally, the documentation has been expanded to include a deep dive into the "Life of a Transaction," providing a step-by-step technical breakdown of how decentralized payments are authorized and executed without sacrificing user privacy. Recent updates to the GitHub repository also show a focus on integrating with the **Model Context Protocol (MCP)**, allowing AI models to interact with payment tools through a standardized interface.
