# Problem Statement: Network-Layer Infrastructure for Autonomous Agent Communication (PILOT)

**Status:** Draft | **Organization:** IETF

## Description
This document describes the requirement for a network-layer infrastructure that provides autonomous agents with first-class network citizenship. It addresses the limitations of current agent communication protocols that rely on centralized application-layer infrastructure and lack native network-layer identity, addressing, and transport.

## Latest Summary
This standard document, "Problem Statement: Network-Layer Infrastructure for Autonomous Agent Communication," identifies a critical gap in the current Internet protocol stack regarding AI agents. While application-layer protocols like the Model Context Protocol (MCP) and Agent-to-Agent (A2A) protocol define how agents communicate, the document argues that these assume the presence of stable, human-centric web infrastructure (DNS, stable IP addresses, and HTTP endpoints). The core problem is that AI agents are often transient, run behind NAT, migrate between hosts, and lack persistent network-layer identities.

The proposed standard seeks to define requirements for a "PILOT" network-layer infrastructure that grants agents "first-class network citizenship." Key issues identified include the coupling of agent identity to infrastructure, the lack of peer-to-peer communication without complex web intermediaries, and the absence of an agent-native trust model (beyond human-centric models like TOFU or expensive CA-based TLS). The document proposes an overlay network approach using virtual addressing to decouple identity from location, similar to the logic of LISP (RFC 9300).

The draft outlines several technical requirements:
* **Virtual Addressing:** Stable identifiers independent of the physical network attachment.
* **NAT Traversal:** Automatic and transparent P2P communication for agents on edge devices.
* **Bilateral Trust:** A model where mutual consent is required before any communication can occur.
* **Privacy-by-Default:** Agents remain invisible until they explicitly authorize a relationship.

## What's New
The most recent development is the release of **draft-teodor-pilot-problem-statement-00** on March 14, 2026. This draft represents a shift in IETF focus from application-layer agent protocols (like MCP/A2A) toward the **network and transport layers**. It introduces the concept of a "PILOT" infrastructure aimed at providing virtual addressing and NAT traversal specifically for AI agents. Notably, the draft claims that approximately 88% of real-world environments involve NAT, making existing HTTP-centric agent communication impractical for edge-based or mobile agents. It also proposes a **privacy-by-default** discovery model to prevent mass enumeration of agent endpoints.
