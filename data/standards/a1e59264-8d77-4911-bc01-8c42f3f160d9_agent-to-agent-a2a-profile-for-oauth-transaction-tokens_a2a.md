# Agent-to-Agent (A2A) Profile for OAuth Transaction Tokens (A2A)

**Status:** Draft | **Organization:** IETF

## Description
Defines a profile for using OAuth Transaction Tokens in Agent-to-Agent (A2A) communication to maintain identity and authorization context across distributed agent workloads. It specifies how call chain context can be embedded within tokens to ensure secure execution flow within trusted domains.

## Latest Summary
The **Agent-to-Agent (A2A) Profile for OAuth Transaction Tokens** is a proposed standards-track specification designed to secure communication between autonomous agents in distributed workloads. Building upon the established [OAuth Transaction Tokens](https://datatracker.ietf.org/doc/draft-ietf-oauth-transaction-tokens/) framework, this profile addresses the unique challenges of agentic systems, such as complex multi-hop call chains, "context rot" (where intent drifts from original authorization), and the need for granular, task-specific permissions.

The profile treats Transaction Tokens as signed, short-lived JWTs that propagate user identity and A2A-specific execution context across trusted domains. Key technical mapping includes using the `subject_token` to carry the original user's OAuth access token, `request_details` to encapsulate immutable "User Input," and `request_context` to handle mutable agent data like "Agent Thinking" or "Status Updates." By utilizing the `purp` (purpose) claim to store unique A2A task identifiers, the standard ensures that agent interactions remain within the scope of the originally authorized transaction. This approach reduces reliance on long-lived credentials and enhances auditability across the entire agent call chain.

## What's New
The standard was officially introduced on **October 20, 2025**, with the publication of `draft-liu-oauth-a2a-profile-00`. This initial draft establishes the rationale for applying OAuth Transaction Tokens to Agent-to-Agent (A2A) protocols. 

Key new concepts introduced in this update include:
*   **Context Mapping**: Direct alignment of A2A Protocol "Message" structures (User Input, Agent Thinking, Metadata) to Transaction Token claims (`tctx` and `rctx`).
*   **Context Rot Mitigation**: Identifying "context rot" as a security risk in agentic systems and proposing signed Transaction Tokens as the primary solution to maintain immutable intent.
*   **Task Identification**: Standardizing the `purp` claim to carry the `a2a.Task.id` for cross-workload traceability.
