# Cross-Domain Interoperability Framework for AI Agent Collaboration (CDI)

**Status:** Emerging | **Organization:** IETF

## Description
A framework for enabling seamless cross-domain interoperability among AI agents operating across different networks, administrative domains, and heterogeneous platforms. It specifies mechanisms for agent discovery, capability negotiation, trust delegation, and federated policy enforcement to facilitate scalable multi-domain AI collaboration.

## Latest Summary
The **Cross-Domain Interoperability Framework for AI Agent Collaboration (CDI)** is a proposed IETF informational standard designed to enable AI agents to collaborate across different administrative domains, networks, and platforms. Traditionally, AI agents are confined to single-domain environments where trust and policies are centrally managed. This framework addresses the growing need for cross-enterprise and cross-border collaboration by providing mechanisms for identity federation, trust establishment without a central authority, and policy harmonization. 

Core architectural components of the framework include:
*   **Interoperability Gateways**: Facilitate communication and policy enforcement between domains.
*   **Agent Registries**: Enable discovery of agent capabilities.
*   **Trust Establishment Protocols**: Defining how domains can federate and delegate authority.
*   **Federated Identity Model**: Utilizing standards like Decentralized Identifiers (DID) and Verifiable Credentials (VC) to manage agent identities.
*   **Capability Discovery**: A standardized language and protocol for agents to negotiate what tasks they can perform for one another.

The framework emphasizes domain autonomy, minimal trust assumptions, and a fail-safe design to ensure that security breaches in one domain do not cascade to others. It also incorporates the **Model Context Protocol (MCP)** for standardizing tool invocation and introduces a **Security Enhancement Layer (SEL)** to provide transparent security controls like data backflow protection and dual authorization.

## What's New
The standard is in its earliest stage with the publication of **version 00** in February 2026. Recent updates focus on integrating the **Model Context Protocol (MCP)** to standardize how agents interact with external tools and hosts. A significant addition is the **Security Enhancement Layer (SEL)**, which introduces "Client Shields" and "Server Shields" to protect user and provider interests without requiring modifications to the underlying protocols. The draft also recently added extensive use cases spanning smart city federation, international research data commons, and global energy grid coordination to justify the multi-domain approach.
