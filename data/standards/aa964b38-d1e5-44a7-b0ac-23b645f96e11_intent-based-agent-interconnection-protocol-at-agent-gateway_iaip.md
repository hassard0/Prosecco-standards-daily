# Intent-based Agent Interconnection Protocol at Agent Gateway (IAIP)

**Status:** Draft | **Organization:** IETF

## Description
A protocol specifying interaction mechanisms between AI Agents and Agent Gateways based on semantic intent rather than static network addressing. It enables agent registration, intent resolution, and dynamic routing based on agent capabilities and task requirements.

## Latest Summary
The **Intent-based Agent Interconnection Protocol (IAIP)** is a proposed application-layer protocol designed to facilitate dynamic, semantic-driven communication between AI Agents and Agent Gateways (AG). Unlike traditional networking protocols that rely on static IP addresses or fixed service identifiers, IAIP focuses on **intent-based routing**. This approach allows agents to be discovered and invoked based on their advertised capabilities and the semantic requirements of a specific task.

The protocol operates within the **Interconnection Service Domain**, acting as a bridge between the **Agent Domain** and the core network infrastructure. It introduces a two-stage processing pipeline:
1.  **Agent Registration & Capability Advertisement:** Agents register their identity to obtain a verifiable Agent Identity Code (AIC) and publish functional profiles (Capability Advertisements) to a Local Capability Registry (LCR).
2.  **Intent-based Routing and Forwarding:** A Leader Agent submits an intent-based request. The Agent Gateway resolves this intent against the LCR using a dynamic routing engine (the Intent Forwarding Routing Table or IFR), selects the most suitable Partner Agent based on semantic matching and performance metrics, and facilitates the interconnection.

Key functional components at the Agent Gateway include the **Agent Access Component (AAC)** for authentication and session management, the **LCR** for storing agent profiles, and the **IFR** for real-time decision-making. The protocol is intended for the IETF Standards Track and aims to solve the limitations of static interconnection in rapidly evolving agentic networks where AI models may be frequently updated or instantiated.

## What's New
The **IAIP-00 draft**, published in **February 2026**, represents the first formal specification of this protocol within the IETF framework. 

Recent additions highlighted in this version include:
*   **Architecture Definition:** Establishment of the three-tier model involving the Leader Agent, Agent Gateway, and Partner Agent.
*   **Functional Components:** Detailed requirements for the **Local Capability Registry (LCR)**, **Agent Access Component (AAC)**, and **Intent Forwarding Routing Table (IFR)**.
*   **Message Types:** Introduction of specific protocol messages such as `ACP-IF-01` (Identity Registration) and `ACP-IF-06` (Capability Advertisement).
*   **Semantic Routing Logic:** A shift toward using "normalized intent vectors" for routing decisions rather than traditional destination addresses, including fallback mechanisms for ambiguous intents.
